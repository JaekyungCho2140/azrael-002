🐱 Azrael 사용자 매뉴얼

L10n(현지화) 일정 관리 도구 — 업데이트 일자 기반 스케줄 자동 계산, JIRA 연동, 이메일 생성

1. 개요

Azrael은 게임 L10n 팀을 위한 일정 관리 도구입니다. 게임 업데이트 날짜를 입력하면, 각 업무 단계의 마감일과 테이블 전달일을 영업일 기준으로 자동 역산하여 3개의 스케줄 테이블, 간트 차트, 캘린더를 생성합니다.

3. 로그인

🌐 Azrael URL 접속 → 🔐 Gmail로 로그인 클릭 → Google 계정 선택 → ✅ 메인 화면
  1. 브라우저에서 Azrael URL에 접속합니다.
  2. "🔐 Gmail로 로그인" 버튼을 클릭합니다.
  3. Google 로그인 팝업에서 회사 계정을 선택합니다.
  4. 화이트리스트에 등록된 계정이면 메인 화면으로 이동합니다.
🚫 접근 거부 시 "접근이 거부되었습니다" 메시지가 표시되면, 관리자에게 화이트리스트 등록을 요청하세요.

4. 최초 접속 (온보딩)

처음 로그인하면 프로젝트 선택 화면이 표시됩니다.

  1. 사용할 기본 프로젝트를 선택합니다 (예: M4/GL, NC/GL 등).
  2. "시작" 버튼을 클릭하면 메인 화면으로 이동합니다.
  3. 온보딩은 한 번만 표시되며, 이후 프로젝트는 헤더에서 변경할 수 있습니다.

6. 일정 계산

업데이트일 입력 → 계산 클릭 → 계산 진행 → 결과 표시
  1. 업데이트일 입력란에 게임 업데이트 예정일을 입력합니다 (예: 2026-02-10).
  2. "계산" 버튼을 클릭합니다.
  3. 시스템이 아래 항목들을 자동 계산합니다:
    • 헤즈업 날짜: 프로젝트 설정의 HeadsUp Offset 영업일만큼 역산
    • iOS 심사일: 프로젝트에서 활성화한 경우만 표시
    • 유료화 상품 협의 일정: 프로젝트에서 활성화한 경우만 표시
    • 3개 스케줄 테이블: 각 업무 단계별 마감일 & 전달일
  4. 결과는 서버(Supabase)에 저장되어 팀원 모두 공유됩니다.
💡 영업일 계산 규칙 토요일, 일요일, 등록된 공휴일은 자동으로 건너뜁니다. 예를 들어 영업일 5일 전은 주말을 제외하고 5일 전입니다.

7. 결과 확인

계산이 완료되면 아래 결과 화면이 표시됩니다.

7.1 스케줄 테이블

3개의 테이블이 각각 다른 대상/목적으로 생성됩니다:

테이블용도색상
테이블 1 (Standard)표준 일정 (담당자 기준)🟠 Orange
테이블 2 (Ext.)외부 파트너용 일정 (JIRA 기준)🟢 Teal
테이블 3 (Int.)내부 팀용 일정 (JIRA 기준)🟣 Purple

컬럼설명

7.2 간트 차트

각 테이블에 대응하는 간트 차트가 표시됩니다. 일 단위 타임라인으로 업무 시작/종료 시점과 의존 관계를 시각적으로 확인할 수 있습니다.

  • 마우스 호버 시 작업명, 시작/종료 시간 팝업 표시
  • 상위-하위 일감 구조가 계층적으로 표현됩니다

7.3 캘린더 뷰

월간 캘린더에 3개 테이블의 일정이 색상별로 오버레이됩니다. 전체 일정을 한눈에 파악하는 데 유용합니다.

8. 이미지 복사

각 테이블과 차트 옆에 📋 복사 버튼이 있습니다.

  1. 복사하고 싶은 테이블/차트의 📋 복사 버튼을 클릭합니다.
  2. 해당 영역이 PNG 이미지로 클립보드에 복사됩니다.
  3. 슬랙, 메일, 문서 등에 Ctrl+V (Mac: ⌘V)로 붙여넣기합니다.
💡 팁 "복사 성공!" 토스트 메시지가 나타나면 정상적으로 복사된 것입니다. 만약 실패하면 브라우저의 클립보드 권한을 확인하세요.

9. JIRA 연결 설정

JIRA 기능을 사용하기 전에 API 토큰 설정이 필요합니다.

⚙️ 설정 → JIRA 탭 → 토큰 입력 → 연결 테스트 → 저장
  1. 헤더의 ⚙️ 설정 버튼을 클릭합니다.
  2. 좌측 사이드바에서 "JIRA 연결" 탭을 선택합니다.
  3. API 토큰 입력란에 JIRA API 토큰을 입력합니다.
    • 토큰 발급: Atlassian 계정 보안 설정
  4. "연결 테스트" 버튼을 클릭하여 유효성을 확인합니다.
  5. 성공 시 계정 이메일과 Account ID가 표시됩니다.
  6. "저장" 버튼을 클릭합니다 (연결 테스트 성공 후에만 활성화).
💡 보안 JIRA API 토큰은 브라우저의 LocalStorage에만 저장됩니다 (서버로 전송되지 않음). 다른 PC에서 사용하려면 해당 PC에서 다시 설정해야 합니다.

10. JIRA 일감 생성

계산 완료 → JIRA 생성 클릭 → 미리보기 확인 → 확인 클릭 → 생성 완료
  1. 일정 계산이 완료된 상태에서 "JIRA 생성" 버튼을 클릭합니다.
  2. 시스템이 자동으로 중복 여부를 확인합니다.
  3. 미리보기 모달에서 생성될 일감 목록을 확인합니다:
    • 📦 Epic: 전체 업데이트 컨테이너
    • 📋 Task: 각 업무 단계 (테이블 2/3 기준)
    • 📄 Subtask: 하위 일감
  4. "확인"을 클릭하면 JIRA에 일감이 생성됩니다.
  5. 완료 시 Epic 키가 포함된 성공 메시지가 표시됩니다.
🚫 중복 생성 방지 동일 프로젝트 + 업데이트일 조합으로 이미 Epic이 존재하면 중복 생성이 차단됩니다. 일정 변경 시에는 "JIRA 업데이트"를 사용하세요.

11. JIRA 일감 업데이트

이미 생성된 JIRA 일감의 일정을 수정할 때 사용합니다.

  1. 업데이트일 또는 업무 단계를 변경한 후 다시 계산합니다.
  2. "JIRA 업데이트" 버튼을 클릭합니다.
  3. 시스템이 기존 Epic 존재 여부를 확인합니다.
    • JIRA에서 수동 삭제된 경우: 정리 후 새로 생성을 안내합니다.
  4. 변경 사항을 확인 후 "확인"을 클릭합니다.
  5. 기존 Task 일정 수정 + 새 Task 생성이 함께 진행됩니다.

12. 이메일 생성 Phase 2

계산 완료 → ✉️ 이메일 복사 → 테이블 선택 → 템플릿 선택 → 클립보드 복사
  1. 일정 계산이 완료된 상태에서 "✉️ 이메일 복사" 버튼을 클릭합니다.
  2. 이메일 생성 모달이 열립니다.
  3. 테이블 선택: 이메일에 포함할 테이블을 선택합니다 (Table 1 / Table 2 / Table 3).
  4. 템플릿 선택: 사용할 이메일 템플릿을 선택합니다.
  5. 미리보기 영역에서 생성된 이메일 내용을 확인합니다.
  6. "클립보드 복사" 버튼을 클릭합니다.
  7. 이메일 클라이언트 (Outlook, Gmail 등)에서 Ctrl+V로 붙여넣기합니다.
💡 HTML 소스 보기 미리보기 영역에서 "HTML 보기" 토글을 켜면 원본 HTML 코드를 확인할 수 있습니다.

14. 프로젝트 관리

⚙️ 설정 → 프로젝트 탭

프로젝트 추가

  1. "+ 새 프로젝트 추가" 버튼을 클릭합니다.
  2. 프로젝트 편집 모달에서 필요한 정보를 입력합니다:
    • 이름 (필수): 프로젝트 표시명
    • HeadsUp Offset: 헤즈업 날짜 역산 영업일 수
    • iOS 심사일 표시: 토글로 활성화/비활성화
    • iOS Review Offset: iOS 심사일 역산 영업일 수 (활성화 시)
    • 유료화 상품 협의 일정 표시: 토글로 활성화/비활성화
    • 유료화 상품 Offset: 유료화 상품 협의 일정 역산 영업일 수 (활성화 시)
    • 면책 조항: 테이블 하단에 표시될 HTML 텍스트 (서식 태그 사용 가능)
  3. JIRA 설정 (선택):
    • 프로젝트 키 (예: L10NM4)
    • Epic/Headsup Summary 템플릿
    • Task Issue Type
  4. "저장"을 클릭합니다.

프로젝트 편집/삭제

  • 편집: 목록에서 프로젝트의 "편집" 버튼 클릭
  • 삭제: 목록에서 "삭제" 버튼 클릭 (확인 대화상자 표시)
⚠️ 마지막 프로젝트 보호 프로젝트가 1개만 남아있으면 삭제할 수 없습니다. 최소 1개의 프로젝트가 필요합니다.

15. 업무 단계 관리

⚙️ 설정 → 업무 단계 탭 → 프로젝트 선택

업무 단계 추가

  1. 상단 드롭다운에서 프로젝트를 선택합니다.
  2. "+ 업무 단계 추가" 버튼을 클릭합니다.
  3. 업무 단계 편집 모달에서 정보를 입력합니다:
    • 이름 (필수): 단계 이름 (예: REGULAR, EXTRA1)
    • 마감 역산 영업일: 업데이트일로부터 역산할 영업일 수
    • 테이블 전달 역산 영업일: 전달일 역산 영업일 수
    • 시작 시간: 기본 09:00
    • 종료 시간: 기본 18:00
    • 대상 테이블: 체크박스로 Table 1/2/3 선택

하위 일감 (Subtask)

업무 단계 편집 모달 하단의 "하위 일감" 아코디언을 열어 하위 업무를 추가할 수 있습니다.

  • 상위 1개 당 최대 9개의 하위 일감
  • 각 하위 일감도 동일한 필드 (이름, offset, 시간, 테이블) 설정 가능
  • 결과 테이블에서 "1.1", "1.2" 형태로 표시

23. 계산 결과 자동 복원 신규

한번 계산한 결과는 자동으로 저장되어, 다음 상황에서 자동 복원됩니다:

  • 프로젝트 전환: 다른 프로젝트로 이동했다가 돌아오면 이전 계산 결과가 자동 로드
  • 페이지 새로고침: F5 또는 브라우저 새로고침 후에도 마지막 계산 결과 유지
  • 앱 재접속: 브라우저를 닫고 다시 열어도 마지막 상태 복원
💡 동작 원리 마지막 계산 날짜는 브라우저(LocalStorage)에, 실제 계산 결과는 서버(Supabase)에 저장됩니다. 따라서 다른 PC에서는 계산 결과가 서버에 있지만 자동 복원되지 않을 수 있습니다 (날짜 정보가 없으므로).

24. 유료화 상품 협의 일정

업데이트일 기준으로 유료화 상품 협의 일정을 자동 역산할 수 있습니다.

설정 방법

  1. 설정 → 프로젝트 관리에서 프로젝트를 선택합니다.
  2. "유료화 상품 협의 일정 표시" 체크박스를 활성화합니다.
  3. Offset 값을 입력합니다 (업데이트일로부터 역산할 영업일 수).
  4. "저장"을 클릭합니다.

결과 확인

  • 메인 화면에서 계산 후, 날짜 요약 섹션에 유료화 상품 협의 일정이 표시됩니다.
  • Disclaimer 텍스트에서 {paidProductDate} 변수를 사용하면 자동 치환됩니다.
  • 이메일 템플릿에서 {{#if showPaidProductDate}} 조건부 블록으로 사용할 수 있습니다.
💡 Disclaimer 변수 프로젝트 면책 조항에서 사용할 수 있는 변수: {updateDate}, {headsUp}, {iosReviewDate}, {paidProductDate}

부록 C. FAQ

Q. 다른 PC에서 로그인했는데 JIRA가 작동하지 않아요.

JIRA API 토큰은 각 PC의 브라우저에 개별 저장됩니다. 새 PC에서는 설정 > JIRA 탭에서 토큰을 다시 설정해야 합니다.

Q. 계산 결과가 이전과 다릅니다.

공휴일 목록이 변경되었거나, 업무 단계 설정(offset)이 수정되었을 수 있습니다. 설정 > 공휴일과 업무 단계를 확인하세요.

Q. 테이블 내용을 직접 수정하고 싶습니다.

스케줄 테이블은 읽기 전용입니다. 설정 > 업무 단계에서 해당 단계의 설명, 담당자 등을 수정한 후 다시 계산하세요.

Q. "JIRA 생성" 버튼이 비활성화 상태입니다.

먼저 일정 계산을 실행하고, 설정 > JIRA에서 토큰 설정 및 프로젝트에 JIRA 프로젝트 키를 입력해야 합니다.

Q. 이메일이 복사되지 않아요.

브라우저에서 클립보드 접근 권한이 차단되었을 수 있습니다. 주소창 왼쪽의 자물쇠 아이콘을 클릭하여 클립보드 권한을 "허용"으로 변경하세요.

Q. 모바일에서도 사용 가능한가요?

아니요. Azrael은 PC 브라우저(최소 1280px) 전용입니다. 모바일/태블릿에서는 정상 작동을 보장하지 않습니다.

Q. 이전에 생성한 JIRA Epic이 삭제되면 어떻게 되나요?

"JIRA 업데이트" 실행 시 시스템이 Epic 존재 여부를 확인합니다. 삭제된 경우 정리 옵션과 함께 새로 생성을 안내합니다.

Q. Slack 연동 시 팝업이 열리지 않아요.

브라우저의 팝업 차단 설정을 확인하세요. Chrome의 경우 주소창 오른쪽에 팝업 차단 아이콘이 표시됩니다. "허용"을 선택한 후 다시 시도하세요.

Q. Slack 메시지가 전송되지 않아요.

다음 항목을 확인하세요: (1) Slack 연동 상태 (설정 > Slack 탭), (2) 채널 매핑 완료 여부, (3) Slack App이 해당 채널에 초대되어 있는지. 비공개 채널의 경우 앱 초대가 필수입니다.

Q. 프로젝트를 전환하면 이전 계산 결과가 사라져요.

Phase 3부터 계산 결과가 자동 복원됩니다. 프로젝트를 전환하면 해당 프로젝트의 마지막 계산 결과가 자동으로 로드됩니다. 만약 복원되지 않으면, 해당 프로젝트에서 한 번이라도 계산을 실행해야 합니다.
