# v1.2 수동 기능 테스트 시나리오

**대상 버전**: v1.2 (2026-02-26)
**배포 URL**: https://azrael-002.vercel.app
**전제 조건**: JIRA API 토큰 설정 완료, 프로젝트에 JIRA 프로젝트 키 설정 완료

---

## 1. Epic Description 설정 UI

### 1-1. Epic Description textarea 표시
1. 설정 > 프로젝트 > 아무 프로젝트 "편집" 클릭
2. "Epic Summary 템플릿" 아래, "헤즈업 Task Summary 템플릿" 위에 **"Epic Description 템플릿 (선택)"** textarea가 표시되는지 확인
3. placeholder 텍스트: "Epic 일감의 설명에 삽입될 내용 (ADF 테이블 마크업 지원)"

**기대 결과**: textarea가 올바른 위치에 표시됨

### 1-2. 표 자동 삽입 토글
1. "스케줄 표 자동 삽입" 체크박스가 textarea 아래에 표시되는지 확인
2. 체크박스 OFF 상태에서는 라디오 버튼이 숨겨져 있는지 확인
3. 체크박스 ON → T1/T2/T3 라디오 버튼 3개 표시되는지 확인
4. 기본 선택값: T1 (Standard)

**기대 결과**: 토글 ON/OFF에 따라 라디오 버튼 표시/숨김 정상 동작

### 1-3. 설정 저장/로드
1. Epic Description에 `{updateDate} 업데이트 일정표` 입력
2. 표 자동 삽입 ON, T2 선택
3. "저장" 클릭
4. 모달 닫고 다시 "편집" 클릭
5. 입력한 값이 그대로 로드되는지 확인

**기대 결과**: 저장한 값 (텍스트, 토글 ON, T2 선택)이 정확히 복원됨

### 1-4. 신규 프로젝트 기본값
1. "+ 새 프로젝트 추가" 클릭
2. Epic Description: 빈 문자열
3. 표 자동 삽입: OFF
4. 테이블 타입: T1 (기본)

**기대 결과**: 신규 프로젝트는 모든 Epic Description 필드가 기본값

## 2. JIRA 생성 — Epic Description 삽입

### 2-1. Epic Description 없이 생성
1. 프로젝트 편집 → Epic Description 비워두기, 표 자동 삽입 OFF → 저장
2. 업데이트일 입력 → 계산
3. "JIRA 생성" 클릭 → 미리보기 확인 → "확인"
4. JIRA에서 생성된 Epic 열기

**기대 결과**: Epic 설명란이 비어있음 (기존 동작과 동일)

### 2-2. Epic Description 텍스트만 삽입
1. 프로젝트 편집 → Epic Description에 `{updateDate} 업데이트 현지화 일정입니다.` 입력, 표 자동 삽입 OFF → 저장
2. 업데이트일 입력 → 계산
3. "JIRA 생성" → 확인
4. JIRA에서 Epic 설명 확인

**기대 결과**: Epic 설명에 `0210 업데이트 현지화 일정입니다.` (변수 치환된 텍스트) 표시

### 2-3. Epic Description + T1 테이블 자동 삽입
1. 프로젝트 편집 → Epic Description에 `[일정표]` 입력, 표 자동 삽입 ON, T1 선택 → 저장
2. 업데이트일 입력 → 계산
3. "JIRA 생성" → 확인
4. JIRA에서 Epic 설명 확인

**기대 결과**: Epic 설명에 "[일정표]" 텍스트 + T1 스케줄 ADF 테이블 (헤더: #, 배치명, 마감, 테이블전달, 설명, 담당자) 표시

### 2-4. Epic Description + T2 테이블 자동 삽입
1. 프로젝트 편집 → 표 자동 삽입 ON, T2 선택 → 저장
2. 계산 → JIRA 생성

**기대 결과**: T2 스케줄 ADF 테이블 (헤더: #, 배치명, HO, HB, 설명) 표시

### 2-5. 표만 삽입 (Description 텍스트 없이)
1. 프로젝트 편집 → Epic Description 비워두기, 표 자동 삽입 ON, T3 선택 → 저장
2. 계산 → JIRA 생성

**기대 결과**: Epic 설명에 T3 스케줄 테이블만 표시 (선행 텍스트 없음)

## 3. JIRA 업데이트 — Epic Description 갱신

### 3-1. Epic Description 업데이트
1. 이전에 생성한 Epic이 있는 상태에서 업데이트일 변경 → 재계산
2. "JIRA 업데이트" 클릭 → 확인
3. JIRA에서 Epic 설명 확인

**기대 결과**: Epic 설명이 새로운 계산 결과로 갱신됨 (변수 값, 테이블 내용 모두 최신)

### 3-2. Epic Description 설정 변경 후 업데이트
1. 프로젝트 편집 → Epic Description 텍스트 변경 또는 테이블 타입 변경 → 저장
2. "JIRA 업데이트" 클릭

**기대 결과**: 변경된 설정이 반영된 Epic Description으로 갱신됨

## 4. 통합 변수 체계

### 4-1. JIRA Summary 변수 치환
1. 프로젝트 편집 → Epic Summary 템플릿: `{updateDate} 업데이트 ({updateDateFull})` → 저장
2. 계산 → JIRA 생성 → 미리보기에서 Epic Summary 확인

**기대 결과**: `0210 업데이트 (26/02/10)` 형태로 표시

### 4-2. Epic Description 변수 치환
1. Epic Description에 다음 입력:
   ```
   업데이트: {updateDateDay}
   헤즈업: {headsUpDay}
   iOS 심사: {iosReviewDateFull}
   ```
2. 계산 → JIRA 생성

**기대 결과**: 각 변수가 올바른 형식으로 치환됨
- `업데이트: 02/10(화)`
- `헤즈업: 01/28(수)`
- `iOS 심사: 26/02/03` (또는 프로젝트 설정에 따라 빈 값)

### 4-3. 레거시 {date} 변수 호환
1. Epic Summary 템플릿: `{date} 업데이트` → 저장
2. 계산 → JIRA 생성

**기대 결과**: `260210 업데이트` (YYMMDD 형식 유지)

### 4-4. 이메일 변수 확인
1. 계산 완료 후 "이메일 복사" 클릭
2. 기본 템플릿 선택 → 미리보기 확인
3. `{updateDate}` → MMDD 형식 표시 확인
4. `{headsUp}` → MMDD 형식 표시 확인 (v1.2 변경: 이전에는 MM/DD(요일))

**기대 결과**: 이메일 미리보기에서 MMDD 형식으로 표시

### 4-5. 이메일 조건부 블록
1. iOS 심사일 활성화된 프로젝트에서 계산
2. 이메일 미리보기에서 `iOS 심사일:` 항목 표시 확인
3. iOS 심사일 비활성화 프로젝트에서는 해당 항목 미표시 확인

**기대 결과**: `{{#if showIosReviewDate}}` 조건부 블록 정상 동작

## 5. 이미지 복사 버튼 제외

### 5-1. 스케줄 테이블 이미지 복사
1. 계산 완료 후 T1 테이블의 "이미지 복사" 버튼 클릭
2. 복사된 이미지를 빈 문서/슬랙에 붙여넣기

**기대 결과**: 복사된 이미지에 "이미지 복사" 버튼 텍스트가 포함되지 않음

### 5-2. 간트 차트 이미지 복사
1. 간트 차트의 "이미지 복사" 버튼 클릭
2. 붙여넣기로 확인

**기대 결과**: 버튼 텍스트 미포함

### 5-3. 캘린더 뷰 이미지 복사
1. 캘린더 뷰의 "이미지 복사" 버튼 클릭
2. 붙여넣기로 확인

**기대 결과**: 버튼 텍스트 미포함

### 5-4. "생성 중..." 상태에서 확인
1. 이미지 복사 버튼 클릭 직후 (생성 중 상태) 다른 테이블의 이미지 복사 실행
2. "생성 중..." 텍스트가 캡처에 포함되지 않는지 확인

**기대 결과**: 어떤 상태에서도 버튼 영역이 캡처에서 제외됨
